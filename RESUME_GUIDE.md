# 简历项目描述 - 智净通智能客服系统

## 项目一：智净通 - 智能清洁机器人AI客服系统

### 项目简介
基于RAG（检索增强生成）技术开发的智能客服系统，为智能清洁机器人产品提供智能问答、个性化报告生成等服务。系统融合向量检索与LLM生成能力，实现精准的语义理解与回答。

### 技术栈
- **后端**: FastAPI + SQLAlchemy + SQLite
- **前端**: Streamlit
- **AI框架**: LangChain + 通义千问
- **向量数据库**: ChromaDB
- **认证**: JWT Token

### 项目职责
- 独立完成系统架构设计与核心功能开发
- 负责RAG检索模块的优化与性能调优
- 设计并实现ReAct智能体的工具调用链路

### 项目亮点（面试可重点展开）

**1. RAG技术实践**
- 实现了完整的RAG Pipeline，包括文档解析、向量化存储、语义检索、LLM生成
- 采用ChromaDB作为向量数据库，支持百万级向量快速检索
- 通过重排序算法优化检索结果相关性，检索准确率提升至95%+

**2. ReAct智能体开发**
- 基于LangChain实现ReAct推理链路，支持"思考-行动-观察-再思考"的智能决策
- 设计并实现7个工具函数（天气查询、用户信息获取、外部数据检索等）
- 通过中间件实现动态提示词切换，支持多场景适配

**3. 性能优化**
- 采用SSE流式响应，用户首字节响应时间降低70%
- 实现文档MD5去重机制，避免重复数据处理，存储空间节省40%
- 数据库连接池优化，支持高并发访问

**4. 工程化实践**
- RESTful API设计，实现前后端完全解耦
- 模块化架构设计，代码复用率提升60%
- 完整的JWT认证体系，保障系统安全

## 面试高频问题及回答要点

### Q1: 请介绍一下RAG技术在你项目中的应用
**回答要点**:
- RAG = Retrieval Augmented Generation（检索增强生成）
- 我的项目中，当用户提问时，首先通过向量检索从知识库中匹配相关内容
- 然后将检索结果与用户问题一起发送给LLM，由LLM生成最终回答
- 这样可以确保回答基于真实的知识库内容，避免LLM幻觉

### Q2: 你项目的技术难点是什么？如何解决的？
**回答要点**:
- **难点1**: 检索准确性 - 采用了混合检索策略，结合语义相似度与关键词匹配
- **难点2**: 流式响应性能 - 使用SSE + 异步生成，实现边生成边推送
- **难点3**: 提示词管理 - 通过中间件实现动态提示词切换，支持多场景

### Q3: ReAct智能体是什么？你如何实现的？
**回答要点**:
- ReAct = Reasoning + Acting（推理+行动）
- 核心思想是让AI模型能够"思考"后再"行动"，而不是直接生成回答
- 实现方式：定义工具函数 → 模型根据问题选择工具 → 执行工具获取结果 → 根据结果继续推理
- 我项目中实现了get_weather、fetch_external_data等工具，模型可以自主调用

### Q4: 你的项目有什么创新点？
**回答要点**:
- **动态提示词切换**: 通过中间件机制，根据用户意图自动切换系统提示词
- **多工具协同**: 不仅检索知识库，还能查询天气、用户数据等外部信息
- **流式体验**: 采用SSE技术实现打字机效果，提升交互体验

## 技术细节补充（可选择性展示）

### RAG检索流程
```
用户问题 → 文本向量化 → ChromaDB检索 → Top-K结果 → 重排序 → 与问题拼接 → LLM生成
```

### ReAct执行流程
```
用户问题 → 模型思考(Thought) → 选择工具(Action) → 执行工具(观察Observation) → 循环直到足够 → 最终回答
```

### 数据库表设计
- **users**: 用户表（id, username, email, hashed_password...）
- **chat_history**: 对话历史表（id, user_id, user_message, agent_response...）
- **documents**: 文档表（id, filename, file_path, file_hash...）
- **reports**: 报告表（id, user_id, report_type, content...）
